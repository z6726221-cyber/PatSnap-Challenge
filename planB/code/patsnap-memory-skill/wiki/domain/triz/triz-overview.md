# TRIZ 是什么

**TRIZ**（俄语缩写，中文常译"发明问题解决理论"或"萃智"）是由苏联发明家 Genrich Altshuller 及其团队从1946年起系统整理的一套发明问题解决方法论。[来源: triz-40-principles, 2024-03-10]

## 核心洞察

TRIZ 的起点是一个反直觉的发现：Altshuller 团队分析了数万份专利后发现，不同技术领域里"看起来完全不同"的发明问题，背后往往对应少数几种通用的解决模式。这意味着一个领域已解决的问题，很可能在另一个领域已经有现成的解法可以借鉴，只是表述方式不同。[来源: triz-40-principles, 2024-03-10]

## TRIZ 的三条主要工具线

TRIZ 不是单一工具，而是一组围绕"如何系统化产生创新方案"目标的工具集合，本知识库覆盖其中三条主线：

1. **技术矛盾 → 矛盾矩阵 → 40个发明原理**：面向"提升A会让B变差"这类矛盾场景。详见 [[triz-contradiction-matrix]] 和 [[triz-40-principles]]。
2. **物场分析 → 标准解**：面向功能结构缺陷（有害作用/缺失作用）场景。详见 [[triz-su-field-analysis]]。
3. **理想最终结果（IFR）**：贯穿全流程的目标导向思维工具，防止过早锁定次优方案。详见 [[triz-ideal-final-result]]。

## 与专利分析的关系

TRIZ 本身就是从专利数据中反向挖掘出来的方法论，这个"从专利反推发明模式"的思路在专利分析工具中也有直接应用，详见 [[triz-in-patent-mining]]。这也是本知识库把 TRIZ 与智慧芽 Engineering Agents 产品能力（[[patsnap-engineering-agents]]）关联起来的原因。

## 待核实

- TRIZ 完整工具体系还包括进化法则、资源分析等本知识库未覆盖的部分，如需完整方法论体系需查阅专门教材，本知识库范围仅限上述三条主线。
