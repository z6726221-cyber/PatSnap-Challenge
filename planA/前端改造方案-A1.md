# 前端改造方案（A1）：统一外壳 + 首页对话 + 功能页锁定 skill

## 目标与共识

把现有 `planA/code/web/index.html`（统一对话入口）升级为「一个应用外壳 + 首页 + 三个功能页」：

- **首页**：约等于现在的对话入口，用户随便问，Agent 走 `run_agent_auto` 自主选 skill；保留跑马灯。
- **功能页**（技术问答 / 竞品分析 / 内容生成）：进入即锁定对应 skill，发送时带 `mode`，后端走 `run_agent` 指定 skill。
- **A1 起步**：功能页复用同一套对话界面，仅改标题 / 副标题 / 输入框占位提示；不做差异化布局（问题 B 竞品表单、问题 C 富文本图表留待后续）。

关键前提：**后端已同时支持两种模式**（`server.py` handle_chat：带 mode → run_agent，不带 → run_agent_auto），因此 chat 逻辑零改动。

## 详细设计

### 1-2. Logo 资源 + 后端静态路由

- 复制 `planA/demo原型/prototype/assets/yado-logo.png` → `planA/code/web/assets/yado-logo.png`（横版 1046×559 RGBA）。
- `server.py` `do_GET` 在 `/api/cases` 分支后新增：对 `/assets/<file>` 读 WEB_DIR 下对应文件返回，带正确 Content-Type（png）。做路径安全校验（禁止 `..` 穿越），只允许 assets 目录内的白名单扩展名。

### 3. 侧栏 logo

`.brand` 由「36×36 深墨方块 + 芽字 + 两行文字」改为单张 `<img class="brand-logo" src="assets/yado-logo.png" alt="芽懂 Yado">`。宽度按 232px 侧栏适配（约 150px），`object-fit:contain` 左对齐。原 slogan「统一口径·带出处」信息已由底部 .foot 卡片承载，不丢失。

### 4. 侧栏功能导航（新增区块）

位置：「＋ 新对话」下方、「历史会话」上方。四个导航项，样式借原型 `.nav-item`（图标+文字+active 高亮），但配色用现有变量（--dark/--green/--side），不引入原型绿。

- 首页（run_agent_auto，Agent 自选）
- 技术问答（mode=qa）
- 竞品分析（mode=comparison）
- 内容生成（mode=promo）

点击 = 切到界面 + 设置 currentMode + 高亮当前项。

### 5. 全局状态

```js
let currentMode = null;   // null=首页 | "qa" | "comparison" | "promo"
```

### 6. send() 带 mode（核心，line 380）

```js
const payload = { message: typed };
if (currentMode) payload.mode = currentMode;
body: JSON.stringify(payload)
```

### 7. 功能页配置表 + 切换

```js
const PAGES = {
  home:       { mode:null,         ttl:"💬 对话",     sub:"随便问，Agent 自动判断该用哪个能力", ph:"输入你的问题…" },
  qa:         { mode:"qa",         ttl:"💬 技术问答", sub:"锁定技术问答，带来源的统一口径答案", ph:"例如：Eureka 到底支持多少种语言？" },
  comparison: { mode:"comparison", ttl:"⚔️ 竞品分析", sub:"锁定竞品对比，按维度对齐出对比表",   ph:"例如：我们 vs Clarivate，帮我对比" },
  promo:      { mode:"promo",      ttl:"✍️ 内容生成", sub:"锁定宣传生成，贴官宣风格带来源初稿", ph:"例如：写一条 Analytics AI Mode 的推文" },
};
```

`goPage(key)`：设 currentMode = PAGES[key].mode，改 .chattop 的 ttl/sub 与 #chatInput 占位，切换视图，高亮侧栏。功能页进入时若无当前 session 则清空 stream 起新对话。

首页仍走 `startFromHome()`（currentMode=null）。

### 8. session 记 mode

`recordMsg` 建 session 时写入 `mode: currentMode`。`openSession` 回看时据 session.mode 恢复 currentMode 与 .chattop 文案、侧栏高亮。兼容旧 session（无 mode 字段按 null 处理）。

### 9. 视觉微调

- 侧栏导航 active 高亮范式借原型。
- 首页保留跑马灯。
- 基调保持现有暖白(--paper)+墨黑(--dark)，不采用原型的墨绿主色，维持辨识度。

## 验证

- `python3 -m unittest discover tests`（后端 30 个测试须仍全过，尤其 server 测试不因静态路由回归）。
- 启动 `python3 server.py`，人工验证：
  1. GET /assets/yado-logo.png 返回图片（200）。
  2. 首页提问 → 不带 mode，Agent 自选（modeflag 显示 Agent 选的能力）。
  3. 点竞品分析 → 提问带 mode=comparison，后端锁定 compare。
  4. 功能页切换时标题/占位提示正确变化。
  5. 刷新后历史会话恢复，点历史条目回到对应界面文案。
  6. 跑马灯、滚动、连续多轮对话不回归。
