# 课题定向 v2：基于本地私域数据的 AI 个性化能力构建

> 本版在 v1（《课题定向.md》）的基础上，**严格对齐官方课题原文**（见《官方课题原文.md》）重写。
> v1 保留不动，作为我们最初的业务化理解；本版作为正式推进依据。
> 校准要点：**能力是主角，数据是样例**——产品是通用的私域知识能力平台，智慧芽的私域数据有一大片（多个产品线、开源项目、论文、历史宣传语料、项目经验）；补齐官方指定的技术交付物（**本地知识库 Skill**）与资源（**Smart-Doc 文档解析 MCP**）；保留 v1 的痛点故事线与"统一口径"增量卖点，但对外统一用官方话术。
>
> **没有唯一主角，只有能力 + 多个示例（关键决策）：** 课题要证明的是**私域知识能力的普适性**——任何智慧芽的私域资料进来，都能建库、问答、生成宣传、竞品对比、路线判断。所以不该有"唯一主角"，而是"能力是主角，数据是并列的样例"。之前反复纠结"选哪个当主角"（先是 TRIZ 理论，后是开源项目）是走偏了。
>
> **关于 TRIZ 的澄清：** 要区分两个东西——**TRIZ 理论本身**是公开通用知识，不是私域数据，不作为素材；但**智慧芽自己做的 TRIZ AI Agent 产品**（其产品能力、技术方案、定位、官宣口径）是智慧芽的**私域产品资产**，可以作为**示例数据集之一**。它是个好讲的样例（小明要推广它、却不懂 TRIZ 也讲不清技术亮点，痛点真实；且天然需要"公开 TRIZ 常识 + 私域产品细节"结合，正好演示"内外数据结合"）。但它只是众多样例之一，不定义产品边界，产品不围绕它做。

## 一、一句话定位

**对内心智（我们自己怎么想）：** 一个"给运营和销售吹牛用的"工具——把技术资料沉淀到本地，业务同学想问什么问什么，还能统一对外技术口径，不用天天追着研发问、也不怕在客户面前露馅。

**对外话术（汇报、企划书怎么说）：** 让通用 AI 从"千人一面"走向"我的 AI、团队的 AI"，记住智慧芽自己的**术语、风格和经验**，让不懂技术的业务同学也能产出专业、可信、符合企业表达的内容与分析。

> 两套说法是同一件事，只是场合不同。对内用来对齐思路，对外用官方定位包装。

---

## 二、官方硬要求（不可偏离的基准）

官方课题白纸黑字规定了四大核心任务和三项期望产出，我们的所有工作必须落在这个框里。

### 四大核心任务

1. **设计本地知识库 Skill，构建智慧芽本地知识库** —— 把论文、开源项目资源及相关本地资料整理为可被 AI 调用的知识库。
2. **为智慧芽开源项目生成富文本宣传材料** —— 结合本地知识库 + 网络数据，体现项目背景、核心能力、技术价值，贴合官宣表达。
3. **分析智慧芽相对竞品的技术优势与不足** —— 本地知识库提取自身能力 + 网络数据了解竞品，形成对比判断。
4. **分析智慧芽在 AI 浪潮中的技术突破路线** —— 找到现有技术能力与 AI 方向的结合点，形成路线判断。

### 三项期望产出

1. **本地知识库 Skill**（核心技术交付物，非泛泛的"向量库"，要做成能被 AI 调用的 Skill/MCP 形态）。
2. **开源项目宣传材料**（结合本地 + 网络数据的富文本，贴合官宣）。
3. **技术对比与路线判断**（竞品技术优劣 + AI 技术突破路线）。

### 官方指定可用资源（必须用上）

- 主流大模型 API + Vibe Coding 工具。
- **Smart-Doc 文档解析 MCP** —— 解析本地文档，把私域资料转成 AI 可处理的内容。（这是官方点名的资源，Demo 里应体现。）
- 智慧芽发表的**论文**和发布的**开源项目**资源 —— 知识库、宣传、技术分析的基础素材。

## 三、v1 → v2 的四处关键校准

1. **能力是主角，数据是并列样例（不设唯一主角）。** 产品是通用的私域知识能力平台；用**智慧芽的多个私域数据集**来演示和验收，样例可包含：智慧芽 TRIZ AI Agent 产品资料、真实开源项目、论文、历史宣传语料等。区分清楚——TRIZ 理论本身是公开知识、不用；智慧芽自研的 TRIZ Agent 产品是私域资产、可作样例之一（详见顶部关键决策）。
2. **"吹牛工具"只做对内心智，对外用官方定位。** 保留这个精准的内部理解，但汇报和企划书统一用"我的 AI、团队的 AI""记住企业术语、风格、经验"这套话术。
3. **补齐技术交付物形态：本地知识库 Skill + Smart-Doc MCP。** 官方期望产出第一项是"Skill"，是明确的技术形态要求。方案和 Demo 必须把这个讲清楚，不能只停留在产品/商业叙事。
4. **"技术突破路线判断"提升为独立任务。** 官方四大任务之一，v1 里被弱化成次要功能，本版恢复其独立地位——虽难但出彩，别丢。

## 四、目标用户

智慧芽**运营、市场、内容策划、产品营销、销售**同学。

## 五、真实痛点（官方三条 + 我们的业务化解读）

官方原文三条：

- **风格漂移**：内容有 AI 味，不像企业自己的表达。
- **术语失准**：识别不了企业/行业/项目内部命名规范，概念易混淆。
- **经验断层**：没结合私域数据，用不上本地文档、报告、翻译记忆、历史经验。

我们的业务化解读（v1 增量，是加分项，可作为讲故事的血肉）：

- **业务不懂技术，什么都问研发** —— 沟通成本高、响应慢。（对应"经验断层"的落地表现）
- **技术口径不一致（最痛）** —— 不同销售说法不同，客户面前易露馅。**本产品相当于统一对外技术口径。**（这是官方没明说的增量卖点，重点讲）
- **横向对比时缺外部弹药** —— 销售天天被客户拿竞品逼问，需要能快速说清"我们的优势在哪"。（对应核心任务 3）

## 六、产品功能（映射到官方任务）

| 功能 | 对应官方核心任务 |
|---|---|
| 私域知识库构建（Smart-Doc 解析 + 向量检索，产出可调用的 Skill） | 任务 1 |
| 专业问答 / 技术口径统一（通俗回答 + 附知识来源） | 任务 1 的应用 |
| 宣传材料生成（公众号、产品介绍、话术、活动推文、案例初稿） | 任务 2 |

| 竞品与技术对比（本地 + 联网 / 用户指定竞品网站） | 任务 3 |
| 技术突破路线判断（自身能力 × AI 趋势结合点） | 任务 4 |

## 七、技术核心（我们自己的判断，官方未细化的部分）

- **数据怎么存、怎么召回是技术核心**：私域知识库 + Smart-Doc 文档解析 + 向量检索 + RAG。
- **回答风格、报告形态由用户 prompt 决定**，不写死。
- **先做一到两个智慧芽私域数据集**（如 TRIZ Agent 产品资料 / 某开源项目 / 论文），围绕它把闭环跑透：知识库 → 问答 → 宣传 → 竞品对比 → 路线判断。做少而透，是为证明能力，不是为堆场景。
- **内外数据结合**：本地私域为主，外部（联网搜索 / 用户抛的竞品网站）为辅。
- **时间线 / 数据更新要处理好**：项目资料会更新，记忆和时间线要能对齐，否则结论会自相矛盾。

## 八、Demo 思路

重点不是功能多，而是**流程完整**：

1. 上传/接入一个智慧芽私域数据集（如 TRIZ Agent 产品资料）→ Smart-Doc 解析 → 建成本地知识库 Skill。
2. 业务同学提问 → 系统通俗回答 + 附来源（体现"统一口径"）。
3. 一键生成宣传富文本（体现"智慧芽风格"）。
4. 抛一个竞品网站/信息 → 输出技术优劣对比 + 宣传切入点（体现"内外数据结合"）。
5. 结合 AI 趋势 → 给出技术突破路线判断。

如来不及做完整系统，可用静态页面 + 预置示例结果模拟以上五步。

---

## 九、汇报重点金句

> 我们不是再做一个普通聊天机器人，而是让 AI 记住智慧芽自己的术语、风格和经验，让不懂技术的业务同学也能快速产出专业、可信、符合企业表达的内容——并且**对外口径统一**。

- 主线：**小明从"不会写、不敢写、口径乱"到"能快速生成专业且一致的内容"**。
- 贴题：**基于本地私域数据的 AI 个性化能力构建**，用智慧芽自己的私域数据（TRIZ Agent 产品、开源项目、论文等）演示这套通用能力。
